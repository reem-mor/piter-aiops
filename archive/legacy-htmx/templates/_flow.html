How it works, step by step

Each step has a plain-English version for non-technical reviewers and a technical note for engineers. Use the for the live request path behind triage and Q&A.

01
Collect runbooks & incident history

Runbooks, alert exports, and post-mortems uploaded to S3 for the Knowledge Base data source.

Technical: PDF / MD / TXT / CSV / DOCX · S3 sync

02
Build the Bedrock Knowledge Base

Bedrock chunks, embeds, and indexes documents so retrieval knows where to look at query time.

Technical: Bedrock KB + vector index · StartIngestionJob

03
Flask triage UI (boto3)

On-call asks a question; Flask calls RetrieveAndGenerate and renders answer, citations, and workflow reasoning.

Technical: POST /ask · POST /workflow/triage · HTMX partials

04
Containerize with Docker

Same image on laptop, CI, and EC2 — no environment drift.

Technical: docker compose · port 8080 · healthcheck

05
Deploy to EC2 & demo

Public URL for reviewers and on-call smoke tests, then cleanup to avoid cost.

Technical: IAM instance profile · security group · screenshots
