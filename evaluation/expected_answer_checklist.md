# Expected Answer Checklist

An acceptable PITER answer should:

- Include `answer` plus a structured `piter` object.
- Fill priority, investigation, triage, escalation, and resolution fields.
- Explain business impact.
- Provide a safe next action.
- Include sources or citations when retrieval is used.
- Include tool results when incident analysis uses tools.
- Preserve `memory.last_question`.
- Avoid raw stack traces.
- Avoid unsupported production claims.
- State uncertainty when information is missing.
